基于改进 YOLOv8 的中医药饮片智能检测与识别系统实践项目任务书

1 项目概述

本项目聚焦中医药药房饮片盘点、仓储管理等实际业务痛点，依托深度学习与计算机视觉技术，开发基于改进 YOLOv8 的中医药饮片智能检测与识别系统。  通用目标检测算法应用于饮片识别场景时，普遍存在小样本数量不足、品类外观相似度高、背景环境复杂干扰强等问题。本项目要求参与者通过网络结构重构、损失函数优化、特征增强设计等方式完成算法升级，实现终端图像中中医药饮片的精准定位与分类识别，并完成端到端工程化部署，形成可落地应用的智能检测解决方案。

2 项目任务与要求

2.1 数据集构建与增强

2.1.1 样本采集

每位成员独立采集 15 种常见中医药饮片，参考品类：桔梗、丹参、板蓝根、白鲜皮、泽泻、桑枝、牛膝、白芍、甘草片、牡丹皮、高良姜、白芷、百合、玉竹、大枣。

硬性要求：数据集必须包含高难度负样本，覆盖复杂桌面背景、纸张 / 包装袋遮挡、光照强弱变化、饮片堆叠、表面污损等真实场景。

2.1.2 数据标注与格式规范

标注工具统一使用 LabelImg，严格遵循 YOLO 标准格式：class_id, x_center, y_center, width, height；标注框需精准贴合目标，杜绝漏标、错标、框体偏移等问题。

数据规模要求：单人完成不少于 20 张高质量图像标注，建议图像分辨率 ≥ 640×480，保障样本特征完整、画面清晰。

2.1.3 数据集划分与增强处理

小组汇总全部样本后统一开展数据清洗，剔除模糊、无效、重复图像；将数据集按照 7:3 或 8:2​ 比例划分为训练集与验证集。

推荐使用 Roboflow、Albumentations​ 工具完成离线数据增强，采用 Mosaic、MixUp、HSV 扰动、随机裁剪​ 等策略，提升模型泛化能力与复杂场景适配性。

2.2 系统功能要求

多目标密集检测：支持单帧图像内多类别、多实例饮片同步检测，可精准区分外观、纹理、形状高度相似的易混淆品类。

端到端实时推理：以原始图像为输入，输出附带检测框、类别标签、置信度的可视化结果；系统需满足实时运行要求，同时兼容 CPU、GPU 两种运行环境。

2.3 算法设计与创新改进

基准模型：基于 Ultralytics YOLOv8 官方框架搭建。

无效改进判定规则：仅调整置信度阈值（conf）、IoU 阈值（iou）、输入尺寸（imgsz）、训练轮数（epochs）等基础超参数，不认定为有效算法改进，按任务未完成处理。

实质性改进方向（任选一项及以上，必须完成代码落地实现）​

（1）骨干网络优化：引入 MobileNetV3、GhostNet 实现模型轻量化；或集成 ConvNeXt 网络，强化细粒度特征提取能力。

（2）注意力机制嵌入：在 Backbone 或 Neck 结构中添加 SE、CBAM、CA（Coordinate Attention）模块，强化饮片纹理、轮廓特征提取，抑制复杂背景干扰。

（3）特征融合增强：将 Neck 层 PANet 替换为 BiFPN（加权双向特征金字塔网络），优化多尺度特征融合效果，提升小目标、堆叠目标的检测精度。

（4）损失函数优化：结合饮片形态细长、品类易混淆的特点，改进 CIoU Loss；或引入 Focal Loss，解决模型难易样本不均衡问题。

（5）检测头优化：采用 Decoupled Head（解耦检测头），拆分分类任务与回归任务，加快模型收敛速度，提升检测精度。

2.4 团队分工与考核机制

分组安排：总人数 20 人，划分为 5 个小组，每组 4 人。各组需明确岗位分工：数据组长、算法核心开发、工程部署、文档撰写。

评优规则：仅参与数据标注、基础参数调试的人员不参与评优；完成核心算法改进，并可提供代码修改记录（Diff）与完整源码的人员，直接获得优先评优资格。

2.5 实践报告与验收标准

2.5.1 报告撰写要求

模型结构图：通过手绘或绘图工具，制作 YOLOv8 改进前后对比图，清晰标注新增、修改模块的位置与功能。

消融实验（必做内容）：展示基准模型（Baseline）与改进模型的 mAP@0.5、mAP@0.5:0.95​ 量化对比数据，结合实验结果定量分析算法改进效果。

2.5.2 现场验收

测试样本：由授课教师提供全新真实饮片样本，包含遮挡、污损、堆叠、复杂背景等干扰场景。

一票否决制：若模型仅能识别规整摆放的样本，在复杂干扰场景下检测失效，直接判定项目不合格。

2.5.3 项目命名规范

报告及项目标题需体现核心技术关键词，参考示例：

《基于 SECBAM 注意力机制的 YOLOv8 中医药饮片检测系统》

《基于 BiFPN 与改进损失函数的轻量化饮片识别算法》

3 开发规范

3.1 开发环境要求

核心框架：Python 3.8+、PyTorch 1.10+、Ultralytics 8.x

工具链：强制使用 Git 管理代码（保留完整 Commit 提交记录）；标注工具使用 LabelImg；开发工具选用 VS Code 或 PyCharm。

算力环境：优先使用 CUDA 11.x GPU 环境；无 GPU 设备需合理调小 batch size，保证模型训练正常进行。
